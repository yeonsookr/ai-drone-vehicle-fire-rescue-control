# 파트별 예외 처리 및 복구 가이드라인 (`exception_guidelines.md`)

본 문서는 산불·조난 대응 드론 및 OrinCar 통합 관제 시스템의 각 레이어(임베디드, 서버, AI, 플랫폼)에서 발생하는 예외 상황을 일관되게 처리하고 안전하게 복구(Fail-safe)하기 위한 공통 가이드라인입니다.

---

## 1. 공통 에러 응답 및 코드 규격

통합 시스템 간 통신(JSON over MQTT / HTTP) 시 에러는 일정한 규격을 따릅니다.

### 1.1 에러 메시지 포맷
```json
{
  "error_code": "ERR_ERRCODE_001",
  "message": "인간이 읽을 수 있는 오류 설명",
  "timestamp": "2026-07-16T15:00:00.000Z",
  "details": {
    "source": "device_id or component_name",
    "reason": "구체적인 기계적/논리적 오류 원인"
  }
}
```

### 1.2 파트별 에러 코드 접두사
- **`EMB_`**: 임베디드 장비(드론, 차량) 및 MQTT Gateway 통신 관련 예외
- **`SRV_`**: 관제 메인 서버, DB, 비즈니스 로직, 안전 정책 게이트 예외
- **`AI_`**: YOLO 추론 지연, 신뢰도 기준 미달, 센서 입력 데이터 불일치 예외
- **`PLT_`**: 웹 관제 화면, 스트리밍 세션, 브라우저 조작 권한 예외

## 2. 파트별 세부 예외 처리 규칙

### 2.1 임베디드 파트 (Embedded Exception Rules)

임베디드 장비는 물리적인 안전(인명 보호, 충돌 방지, 기체 추락 방지)을 최우선으로 합니다.

| 예외 상황 | 탐지 방식 | Fail-Safe 대응 동작 (즉시 복구) |
|---|---|---|
| **드론-차량 통신 단절** | Heartbeat 패킷 누락 (3초 초과) | 비행을 즉시 멈추고 제자리 호버링(Hover)을 유지하며, 추가 통신 시도가 없을 경우 차량의 최종 위치로 **자율 복귀(RTL)** 또는 **안전 착륙**을 시작합니다. |
| **드론 배터리 임계치 도달** | 배터리 잔량 모니터링 (예: 20% 이하) | 순찰 임무를 강제 중지하고, 현재 위치에서 안전하게 복귀하는 데 필요한 배터리 소모량을 계산하여 **긴급 회차** 명령을 발동합니다. |
| **차량 주행 중 장애물 감지** | LiDAR 센서 실시간 거리 판정 | 모터 제어 보드(PCA9685)의 주행 신호를 차단(watchdog 작동)하고 **즉시 물리적 브레이크**를 가해 비상 정지합니다. |
| **명령 유효시간 만료** | 수신된 `commands`의 `expires_at` 만료 | 차량 Orin은 만료된 제어 명령의 전송을 거부하고 서버로 `status=EXPIRED`를 회신합니다. |

### 2.2 관제 서버 파트 (Server Exception Rules)

서버 파트는 전체 비즈니스 정합성과 다중 관제 시 상태의 무결성을 보호합니다.

- **데이터베이스 트랜잭션 예외**:
  - 임무 상태 변경(`missions`)과 이에 따른 하위 제어 명령(`commands`) 발행 실패 시, 전체 상태를 `ROLLBACK`하고 로그 디렉토리에 추적 정보와 에러를 로깅합니다.
- **2구간 중계 메시지 유실**:
  - 서버가 명령을 내렸으나 차량(Gateway)으로부터 ACK를 수신하지 못한 경우, 3회 재시도(Retry) 후 최종 실패(`status=FAILED`) 처리하고 관제 플랫폼에 경보 이벤트를 전송합니다.
- **인가되지 않은 제어 명령**:
  - 역할 기반 권한(RBAC) 검증에 실패한 사용자의 제어 API 요청은 즉시 차단하고 `403 Forbidden` 처리와 함께 `audit_logs` 테이블에 침입 시도(IP, 계정)를 로깅합니다.

## 3.3 AI 파트 (AI Inference Exception Rules)

AI 파트는 오탐(False Alarm)과 노이즈, 불완전한 현장 입력으로부터 시스템의 판단 신뢰성을 유지합니다.

- **낮은 신뢰도(Confidence) 탐지**:
  - YOLO 모델의 탐지 신뢰도가 설정값(예: 0.5) 미만인 경우, 자동 비상 경보를 울리지 않고 관제 플랫폼에 `operator_judgment=pending`(보류) 상태로 전송하여 사람의 직접 검토를 대기합니다.
- **카메라 프레임 유실 및 노이즈**:
  - 드론 전송 영상의 끊김 또는 손상이 연속 5프레임 이상 지속될 경우, 추론 모듈은 추론을 일시 중단하고 `AI_FRAME_LOSS` 이벤트를 발생시켜 차량 Gateway로 안전 복귀 의사결정을 유도합니다.
- **입력 데이터 강건성 불일치 (센서 교차 검증)**:
  - GPS 위도/경도가 궤적에서 급격하게 이탈하거나 센서 정보와 영상 분석 결과가 불일치하는 경우, 의심 입력(`AI-006`)으로 분류하고 해당 모델 판단에 따른 자율 제어 행위를 전면 불허합니다.

## 2.4 플랫폼 파트 (Platform Exception Rules)

플랫폼 파트는 실시간 화면에서 조작자가 상황을 왜곡되게 보거나 오판하지 않도록 돕습니다.

- **WebSocket 연결 끊김**:
  - WebSocket 끊김 시 즉시 화면에 경고 배너를 노출하고 5초 주기로 지수 백오프(Exponential Backoff) 재연결을 시도합니다. 연결 전까지 장비 제어 조작 버튼을 비활성화(Disable) 처리합니다.
- **영상 스트림 송출 장애**:
  - 드론 비디오 스트림 수신 실패 시 검은 화면에 재생 오류 메시지 및 마지막 정상 수신 스냅샷을 촬영 장소 메타데이터와 함께 표시합니다.
- **중요 경보 오버레이**:
  - 산불 또는 조난자 발견(`Critical` 경보)은 다른 일반 로그 화면보다 무조건 최상위(Z-index) 레이어에 노출되며, 조작자가 확인 버튼을 누르기 전까지는 닫히지 않습니다.
