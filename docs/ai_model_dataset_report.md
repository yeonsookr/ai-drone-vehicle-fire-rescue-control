# AI 모델 학습용 데이터 탐색 및 선정 결과

| 항목 | 내용 |
|---|---|
| 대상 프로젝트 | 산불·조난 대응 드론·OrinCar 통합 관제 시스템 |
| 문서 목적 | AI 학습·검증 데이터의 탐색, 선정, 품질 검증, 전처리 및 분할 기준 정의 |
| MVP 대상 | Jetson Orin 기반 산불·연기 및 조난자 객체 탐지 |
| 확장 대상 | 독립 GPU AI 서버 기반 심화 검증 및 산불 확산 예측 |
| 주요 관련 기능 | `AI-001`~`AI-004`, `AI-006`, `AI-007`, `OBS-002`, `SEC-005` |



## 1. 목적과 선정 원칙

1. MVP의 `AI-001`, `AI-002` 객체 탐지 데이터를 가장 먼저 확보한다.
2. 실제 운영 조건과 유사한 드론·원거리·산악 환경 데이터를 우선한다.
3. 학습용 데이터와 독립 검증용 데이터를 분리하고, 같은 영상이나 연속 프레임이 서로 다른 분할에 섞이지 않게 한다.
4. 정확도뿐 아니라 Jetson Orin에서의 추론 지연, FPS, 메모리 사용량을 함께 평가한다.
5. 낮은 신뢰도와 입력 이상은 자동 고위험 명령으로 이어지지 않도록 별도 실패·강건성 데이터로 검증한다.
6. 출처, 라이선스, 수집·가공 이력, 데이터 버전을 추적할 수 없는 데이터는 사용하지 않는다.
7. 조난자 영상은 개인정보와 초상권을 고려해 최소 수집·비식별화·접근 통제 기준을 먼저 확정한다.

`AI-005` 드론·차량 경로 계획은 기능명세서상 Backend의 결정적 알고리즘이 주 책임이다. 산불 위험도 데이터는 선택적 입력으로 활용할 수 있지만, 본 문서의 1차 학습 데이터 선정 범위에서는 제외한다.

`YOLO26n`는 엣지 추론용 `YOLO26x`, `SAM3`은 서버 재검증용 모델 후보이다. 최종 모델·버전과 Orin/서버 간 책임 분리는 `D-013`, `D-015` 의사결정에서 확정한다.

## 3. 데이터 요구사항
이 항목의 내용은 가이드라인이며, 짧은 프로젝트 기간이라는 한계 상 추후 간소화/변경 가능성이 농후하다.
### 3.1 객체 탐지 데이터

| 구분 | 요구사항 |
|---|---|
| 기본 형식 | 이미지 또는 영상 프레임과 YOLO/COCO 등 변환 가능한 경계 상자 라벨 |
| 필수 클래스 | `forest_fire`, `smoke`, `distressed_person` |
| 필수 라벨 | 클래스, 경계 상자 좌표, 이미지 크기, 라벨 검수 상태 |
| 권장 메타데이터 | 촬영 시각, 위도·경도·고도, 카메라 종류·해상도·화각, 드론 거리·고도, 주·야간, 날씨, 출처 |
| 운영 연계 필드 | `mission_id`, `event_id`, `drone_id`, `vehicle_id`, `model_version`, `detected_at` |
| 필수 부정(negative) 샘플 | 우선 확보: 구름, 굴뚝 연기, 안개·연무. 추가 후보: 불·연기·사람이 없는 산림, 수증기·먼지·일몰, 캠프파이어·차량 불빛, 바위·나무·시설물 |
| 필수 난이도 구간 | 원거리 소형 객체, 연기 초기 단계, 부분 가림, 역광, 저조도·야간, 흔들림·압축 손상, 다양한 계절·지형 |

운영 이벤트의 클래스와 출력 형식은 `ai_detections` 스키마에 맞춘다.

```json
{
  "detection_type": "forest_fire | smoke | distressed_person",
  "confidence": 0.0,
  "bounding_box": { "x": 0, "y": 0, "w": 0, "h": 0 },
  "latitude": 0.0,
  "longitude": 0.0,
  "altitude": 0.0,
  "model_version": "model-name-version"
}
```

학습 데이터에 위치 정보가 없더라도 객체 탐지 학습에는 사용할 수 있다. 다만 E2E 검증 데이터에는 촬영 시각·위치·고도와 장비 식별자를 포함해 `OBS-002`, `AI-003` 흐름을 검증할 수 있어야 한다.

### 3.2 산불 확산 예측 데이터

| 범주 | 필요한 변수 | 검토 사항 |
|---|---|---|
| 발화·확산 이력 | 발화 위치·시각, 시간대별 화선 또는 피해 영역, 진화 시각 | 동일 사건의 시간 순서와 좌표계 보존 |
| 기상 | 풍향, 풍속, 습도, 기온, 강수, 관측·예보 시각 | 관측 지점과 산불 위치 간 거리, 시간 해상도 |
| 지형 | 고도, 경사도, 방위, 협곡·능선, 수계 | 좌표계, 공간 해상도, 결측 영역 |
| 수목·연료 | 수종, 임상, 낙엽·연료 상태, 토양·연료 수분 | 갱신 주기와 실제 사건 시점의 일치 여부 |
| 정답·출력 | 시간대별 확산 영역과 위험도 | `GeoJSON Polygon` 변환 가능 여부, 불확실성 표기 |

`fire_predictions` 스키마의 최소 입력은 `weather_wind_speed`, `weather_wind_direction`, `weather_humidity`, `weather_temperature`이며, 결과는 지도에 표시할 `prediction_polygon`과 `model_version`을 기록해야 한다.

## 6. 기초 탐색 분석 및 품질 검증
이 항목은 참고용으로 작성되었다.
### 6.1 공통 점검

- [ ] 파일 수, 전체 용량, 이미지·영상 해상도와 포맷을 확인한다.
- [ ] 손상 파일, 결측 메타데이터, 중복·유사 샘플 비율을 확인한다.
- [ ] 데이터셋별 클래스명과 라벨 형식을 공통 스키마로 매핑한다.
- [ ] 데이터 출처, 버전, 다운로드·수집일, 체크섬을 기록한다.
- [ ] 실제 데이터와 `mock`, `replay` 데이터를 구분한다.
- [ ] 무작위 샘플과 오류 가능성이 높은 샘플을 사람이 직접 검토한다.

### 6.2 객체 탐지 점검

- [ ] 클래스별 이미지 수와 객체 수를 각각 집계한다.
- [ ] 경계 상자의 크기·가로세로 비율·화면 위치 분포를 확인한다.
- [ ] 빈 라벨, 범위를 벗어난 좌표, 잘못된 클래스, 지나치게 작거나 큰 상자를 탐지한다.
- [ ] 같은 영상의 인접 프레임과 유사 이미지가 과도하게 포함됐는지 확인한다.
- [ ] 주·야간, 계절, 날씨, 거리, 고도, 카메라별 분포를 확인한다.
- [ ] 안개·구름·수증기·먼지·일몰 등 산불·연기 오탐 유발 샘플을 별도 집계한다.
- [ ] 사람 라벨의 누락·부분 가림·소형 객체 품질과 개인정보 포함 여부를 확인한다.
- [ ] 일부 표본을 이중 라벨링해 라벨러 간 일치도와 경계 상자 IoU를 확인한다.

### 6.3 확산 예측 점검

- [ ] 사건별 발화·확산·진화 시각이 시간 순서대로 연결되는지 확인한다.
- [ ] 모든 공간 데이터의 좌표계와 해상도를 기록하고 변환 오차를 검증한다.
- [ ] 기상 관측값의 단위, 관측 지점, 시간대, 결측·이상치를 확인한다.
- [ ] 지형·수목 데이터의 제작·갱신 시점이 산불 사건 시점과 맞는지 확인한다.
- [ ] 동일 산불 사건이 학습·검증·테스트에 나뉘어 들어가지 않도록 사건 ID를 구성한다.
- [ ] 실제 확산 영역을 `GeoJSON Polygon`으로 변환할 수 있는지 확인한다.

## 8. 모델 평가와 수용 기준

정량 합격값은 `D-005`에서 AI와 SI·QA가 향후 합의한다.
아래는 참고사항이다.

### 8.1 객체 탐지

- Precision, Recall, F1-score
- 클래스별 AP와 전체 `mAP@0.5`, `mAP@0.5:0.95`
- 영상·비행 시간 기준 오탐률과 미탐률
- 소형·원거리 객체, 주·야간, 날씨, 카메라별 세부 성능
- 빈 프레임·손상 프레임·센서 불일치 입력에 대한 거부·보류 비율
- Jetson Orin 실장 환경의 지연 시간, FPS, GPU·메모리 사용량

### 8.2 산불 확산 예측

- 예측 영역과 실제 영역의 IoU 또는 Dice 계수
- 시간대별 확산 면적 오차와 화선 위치 오차
- 주요 지점 도달 시간 오차
- 입력 결측·오차에 따른 민감도와 불확실성
- 지역·계절·산불 규모별 성능 편차

### 8.3 통합 검증

- AI 결과에 종류, 신뢰도, 경계 상자, 위치, 모델 버전, 근거 프레임이 포함되는지 확인한다.
- 낮은 신뢰도 결과가 `operator_judgment=pending`으로 전달되고 자동 고위험 명령을 발생시키지 않는지 확인한다.
- 빈 프레임, 5프레임 이상 연속 손상, 잘못된 GPS, 센서 불일치가 `AI-006`의 의심·거부 흐름으로 처리되는지 확인한다.
- Orin 추론 실패 시 원본 데이터와 실패 상태가 서버에 남고 보수적 대기·사용자 검토로 전환되는지 확인한다.
- 모델·데이터·임계값 버전으로 동일 시험을 재현할 수 있는지 확인한다.
